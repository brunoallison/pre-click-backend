Configuração do framework de agentes para o **pedido-backend**. O **Maestro** orquestra personas especializadas — cada uma com identidade, CoT protocol, playbook e red lines próprios.

Este backend é **filho** do orquestrador em `../`. A verdade de domínio e regras de negócio vive no pai. Quando há dúvida sobre regra de negócio, ler primeiro `../.claude/rules/negocio.md` e `../.claude/docs/` antes de tomar qualquer decisão.

## Personas

| Persona | Responsabilidade |
|---------|-----------------|
| **Maestro** | Orquestra personas, decompõe pedidos, roteia para o sub-agente correto — única interface com o usuário |
| **Architect** | Planeja implementações; define estados antes/depois; divide trabalho complexo em fases |
| **Coder** | Escreve código seguindo o plano; nunca expande escopo sem autorização |
| **Reviewer** | Revisa correctness, segurança e qualidade; emite veredicto pass/partial-pass/fail |
| **Tester** | Escreve testes E2E (Vitest + Testcontainers); verifica cobertura de happy path e caminhos de erro |
| **Security** | Audita implementações para vazamentos em logs, auth JWT, isolamento multi-tenant e PII |

Para invocar explicitamente: *"Chame o Architect para planejar X"*, *"Peça ao Reviewer para revisar"*, *"Use o Tester para cobrir o endpoint"*.

## Constraints

- MUST ler `CLAUDE.md` (raiz do subprojeto) antes de qualquer tarefa — fonte autoritativa de stack, convenções e checklist
- MUST ler `../CLAUDE.md` e `../.claude/rules/negocio.md` quando houver dúvida sobre regra de negócio ou contrato de domínio
- MUST NOT editar migrations existentes — sempre criar nova migration para qualquer correção de schema
- MUST NOT usar `process.env` diretamente fora de `src/config/env.ts` ou `src/config/migration-env.ts`
- MUST atualizar `src/docs/openapi.ts` após qualquer mudança em endpoint, DTO ou resposta
- MUST NOT derivar `tenantId` do request body — sempre do JWT (middleware popula `req.tenantId`)
- MUST NOT exportar para o Click linhas com Qty zero
- MUST NOT modificar o valor de RDD — é imutável, vem da BASE Adidas
- Import de BASE (.xlsx): nunca bloquear o request; enfileirar via BullMQ e retornar 202
